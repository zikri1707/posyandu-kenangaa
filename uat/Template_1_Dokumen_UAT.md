## Proses Pengujian

| No | Use Case | Hasil Uji [Berhasil\|Gagal] | Nama Penguji * | Tanggal Pengujian | Catatan Penguji |
|:---:|:---|:---:|:---:|:---:|:---|
| **1** | **Usecase Uji** : UAT-01 (Otentikasi & Login Multi-role)<br>**Deskripsi** : Menguji proses login ke sistem menggunakan berbagai akun role (Superadmin, Admin, Kader) untuk memastikan akses menu sesuai hak otorisasi.<br><br>**Kasus Pengujian** :<br>- *Username* : `admin@posyandu.local` (Superadmin) / `kader@posyandu.local` (Kader)<br>- *Password* : `password`<br>- *Chapcha* : `-` (Tidak aktif / bypass)<br><br>**Hasil yang diharapkan** :<br>- Pengguna berhasil login ke dalam sistem.<br>- Pengguna diarahkan ke Dashboard yang sesuai dengan role masing-masing.<br>- Sidebar menampilkan menu sesuai hak akses (misal: menu *Settings* & *User Management* hanya terlihat oleh Superadmin). | [ ] Berhasil<br>[ ] Gagal | | | |
| **2** | **Usecase Uji** : UAT-02 (Keamanan Login - Throttling Brute Force)<br>**Deskripsi** : Menguji ketahanan sistem terhadap serangan brute force dengan memasukkan password salah berulang kali.<br><br>**Kasus Pengujian** :<br>- *Username* : `admin@posyandu.local`<br>- *Password* : `salahpassword123` (diuji 5 kali berturut-turut)<br>- *Chapcha* : `-`<br><br>**Hasil yang diharapkan** :<br>- Percobaan login ke-1 hingga ke-5 gagal dengan pesan "Kredensial tidak cocok".<br>- Pada percobaan ke-6, sistem otomatis mengunci login (throttle) dan menampilkan pesan error rate-limit (misal: "Terlalu banyak percobaan login. Silakan coba lagi dalam 60 detik."). | [ ] Berhasil<br>[ ] Gagal | | | |
| **3** | **Usecase Uji** : UAT-03 (Bulan Penimbangan - Muat Daftar Balita Massal)<br>**Deskripsi** : Menguji fungsi memuat seluruh data balita pada posyandu dan tanggal tertentu untuk penimbangan massal.<br><br>**Kasus Pengujian** :<br>- *Tanggal Kunjungan* : `2026-06-30`<br>- *Posyandu* : `Kenanga 1`<br>- *Aksi* : Klik tombol "Muat Semua Balita"<br><br>**Hasil yang diharapkan** :<br>- Sistem memproses data dan memuat seluruh daftar balita yang terdaftar di bawah Posyandu Kenanga 1 pada tabel pengisian.<br>- Baris tabel menampilkan nama balita, umur (bulan), serta input kosong untuk Berat Badan (BB) dan Tinggi Badan (TB). | [ ] Berhasil<br>[ ] Gagal | | | |
| **4** | **Usecase Uji** : UAT-04 (Bulan Penimbangan - Input & Validasi BB/TB)<br>**Deskripsi** : Menguji validasi input nilai Berat Badan (BB) dan Tinggi Badan (TB) balita agar mencegah kesalahan data (minus/non-numerik).<br><br>**Kasus Pengujian** :<br>- *Input BB valid* : `12.5` kg | *Input TB valid* : `85.2` cm<br>- *Input BB tidak valid* : `-5` kg atau `abc` | *Input TB tidak valid* : `-80` cm atau `xyz`<br><br>**Hasil yang diharapkan** :<br>- Input valid diterima, kolom berubah hijau/menyimpan sementara, dan *progress bar* keterisian bertambah secara dinamis.<br>- Input minus atau non-numerik otomatis diblokir sistem, menampilkan tanda merah/pesan validasi "Format tidak valid" atau "Nilai tidak boleh negatif". | [ ] Berhasil<br>[ ] Gagal | | | |
| **5** | **Usecase Uji** : UAT-05 (Bulan Penimbangan - Penyimpanan Massal)<br>**Deskripsi** : Menguji penyimpanan data penimbangan massal dan kalkulasi status gizi otomatis.<br><br>**Kasus Pengujian** :<br>- *Data BB/TB Balita* : Terisi lengkap (misal: Balita A: BB=10.5 kg, TB=80 cm)<br>- *Aksi* : Klik tombol "Simpan Data"<br><br>**Hasil yang diharapkan** :<br>- Sistem menyimpan seluruh data pengukuran ke tabel rekam medis (`medical_records`) di database.<br>- Muncul notifikasi sukses "Data penimbangan berhasil disimpan".<br>- Sistem menghitung otomatis status gizi balita (BB/U, TB/U, BB/TB) berdasarkan standar WHO dan menyimpannya di baris rekam medis tersebut. | [ ] Berhasil<br>[ ] Gagal | | | |
| **6** | **Usecase Uji** : UAT-06 (Manajemen Pasien - Registrasi Pasien Balita)<br>**Deskripsi** : Menguji penambahan data pasien balita baru secara manual ke dalam sistem.<br><br>**Kasus Pengujian** :<br>- *NIK* : `3404123456789012` (16 digit numerik)<br>- *Nama* : `Ahmad Fauzi`<br>- *Tanggal Lahir* : `2024-01-15`<br>- *Jenis Kelamin* : `Laki-laki`<br>- *Orang Tua (Ibu/Ayah)* : `Siti Aminah` / `Budi Luhur`<br>- *Pedukuhan* : `Kenanga`<br><br>**Hasil yang diharapkan** :<br>- Data balita berhasil disimpan ke database.<br>- Sistem otomatis mengarahkan ke halaman daftar pasien dan menampilkan pesan sukses.<br>- Nama "Ahmad Fauzi" muncul di daftar pasien dan grafiknya siap untuk diakses. | [ ] Berhasil<br>[ ] Gagal | | | |
| **7** | **Usecase Uji** : UAT-07 (Manajemen Pasien - Import Massal via Excel)<br>**Deskripsi** : Menguji fungsi import data pasien secara massal menggunakan file Excel (.xlsx) sesuai template.<br><br>**Kasus Pengujian** :<br>- *File Excel* : Mengunggah file template valid `template_balita.xlsx` yang sudah terisi data balita.<br>- *Aksi* : Klik tombol "Import Data"<br><br>**Hasil yang diharapkan** :<br>- File Excel berhasil diunggah dan diuraikan (parsed) oleh sistem.<br>- Seluruh baris balita dalam file Excel berhasil disimpan ke database.<br>- Tampil pesan sukses "Berhasil mengimpor X data balita". | [ ] Berhasil<br>[ ] Gagal | | | |
| **8** | **Usecase Uji** : UAT-08 (Pencatatan Rekam Medis Individual)<br>**Deskripsi** : Menguji pencatatan rekam medis lengkap untuk seorang pasien secara individu oleh petugas kesehatan.<br><br>**Kasus Pengujian** :<br>- *Pasien* : Pilih `Ahmad Fauzi`<br>- *Berat/Tinggi Badan* : `11.2` kg / `82` cm<br>- *Lingkar Kepala/Lengan* : `46` cm / `14` cm<br>- *Suhu/Keluhan* : `36.5` °C / `Batuk ringan`<br>- *Diagnosa/Terapi* : `Sehat` / `Pemberian Vitamin A & Edukasi Gizi`<br><br>**Hasil yang diharapkan** :<br>- Rekam medis individu berhasil disimpan.<br>- Sistem menghitung status gizi pertumbuhan anak dan menampilkannya di rekam medis.<br>- Data rekam medis masuk ke tabel riwayat pasien dan memperbarui diagram grafik pertumbuhan anak. | [ ] Berhasil<br>[ ] Gagal | | | |
| **9** | **Usecase Uji** : UAT-09 (Audit Trail - Pemantauan Log Aktivitas)<br>**Deskripsi** : Menguji sistem pencatatan log aktivitas untuk mengawasi setiap tindakan CRUD yang dilakukan pengguna.<br><br>**Kasus Pengujian** :<br>- *Aksi Pemicu* : Superadmin membuat user baru atau mengubah data pasien.<br>- *Langkah Pengujian* : Buka menu "Log Aktivitas" (`/admin/activity-logs`) menggunakan akun Superadmin.<br><br>**Hasil yang diharapkan** :<br>- Aksi pembuatan user atau pengubahan pasien tercatat pada baris log teratas.<br>- Log mencantumkan informasi lengkap: Aktor (Superadmin), Modul (User/Patient), Event (created/updated), IP Address, dan Detail Data (before & after).<br>- Properti sensitif seperti password disamarkan (`*****`) demi keamanan data. | [ ] Berhasil<br>[ ] Gagal | | | |
| **10** | **Usecase Uji** : UAT-10 (Laporan & Ekspor Data Posyandu)<br>**Deskripsi** : Menguji penarikan laporan rekapitulasi bulanan posyandu serta ekspor ke format dokumen PDF dan Excel.<br><br>**Kasus Pengujian** :<br>- *Filter Posyandu* : `Kenanga 1`<br>- *Filter Periode* : Bulan `Juni`, Tahun `2026`<br>- *Aksi* : Klik tombol "Ekspor PDF" atau "Ekspor Excel"<br><br>**Hasil yang diharapkan** :<br>- Sistem memproses data rekapitulasi penimbangan balita pada Posyandu Kenanga 1 periode Juni 2026.<br>- File berformat PDF atau Excel (.xlsx) terunduh secara otomatis.<br>- Dokumen yang dihasilkan menampilkan data rekap yang rapi, berisi daftar balita beserta BB, TB, Status Gizi, dan tanda tangan ketua kader. | [ ] Berhasil<br>[ ] Gagal | | | |
